Right click a new folder in your desktop and open git bash. Change Directory into the folder, like so:

$ cd /nameofproject

Create an isolated environment with the following command:

$ python -m venv my_env

Run the following command to activate your virtual environment try:

$ source ./my_env/Scripts/activate

or for iOS or Linux terminals

$ source ./my_env/bin/activate

In the terminal, round brackets now encapsulate the venv you set up:

(my_env)

** It is possible to follow the file path to the /activate file command within the virtualenv folder. This *venv*, creates an isolated environment for python packages (including Django) and it is located inside your main nameofproject folder, where the django_site folder will soon be located.**

$ pip list

pip list will show what requirements are installed into the venv library just created in the project level folder. To deactive the venv

(my_env)

$ deactivate

With your environment **activated**, run the following command in your terminal to install Django (nested in your newly created environment) with pip:

$ pip install "Django==3.2.2"

pip is a recursive acronym 'pip install python' Now enter in

$ pip list

again to reveal Django== and a couple other modules have been added to the virtual environment.

$ django-admin startproject django_site

Change Directory into main django_site.

$ cd django_site/

List files in that directory

$ ls

Notice Django made two directories with that name. Let's now add a single app that will appear in this same directory, along with the inner 
django_site
folder.

$ django-admin startapp mynewapp

*Not yet, but make sure additional apps don't have duplicate names*

$ django-admin startapp AddAnotherApp

**Hint: The mynewapp *folder/directory* does not contain a settings.py file. settings.py is found in the inner "app - level" django_site folder with wsgi.py and asgi.py.

runserver

In the same outer django_site directory, with the manage.py, db.sqlite3, django_site, and mynewapp folders, enter in the terminal

$ python manage.py runserver

This automatically starts a development server in your local browser at:

127.0.0.1:8000/

If you see an animated green rocketship you have success! Now type ctrl+c to break out of the development server

We need to tell Django where we are storing the static and media files while python manage.py runserver are running in development mode on localhost. In the same settings.py, bottom of the page on a blank line enter the following code: 
STATIC_URL = '/static/'

and add these two lines to access media files in development mode:

MEDIA_ROOT = BASE_DIR, 'media'
MEDIA_URL ='/media/'
Add the following line of code to the top of urls.py file in the main project directory (found directly under the settings.py file)

from django.conf.urls.static import static from django.conf import settings

Under the urlpatterns = [...], in the same urls.py file, add the following

settings.DEBUG:
     urlpatterns += static(settings.MEDIA_URL, 
             document_root=settings.MEDIA_ROOT)

A class Field creates a table in the model. A table that contains the preset values for multiple fields.
To get models to appear on the Admin page locate the admin.py in the app folder. 

from django.contrib import admin
from .models import Post, Comment
@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'slug', 'author', 'publish', 'status')
    list_filter = ('status', 'created', 'publish', 'author')
    search_fields = ('title', 'body')
    prepopulated_fields = {'slug': ('title',)}
    raw_id_fields = ('author',)
    date_hierarchy = 'publish'
    ordering = ('status', 'publish')


@admin.register(Comment)
class CommentAdmin(admin.ModelAdmin):
    list_display = ('name', 'email', 'post', 'created', 'active')
    list_filter = ('active', 'created', 'updated')
    search_fields = ('name', 'email', 'body')
