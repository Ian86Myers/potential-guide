Open a new folder in your desktop and rename it DjangoGuide, right click on the folder/directory and open Git Bash. Or open Git Bash and type in the following to make a new directory:

$ mkdir DjangoGuide

Change Directory into the folder, like so:

$ cd /DjangoGuide

Create an isolated environment inside DjangoGuide with the following command:

$ python -m venv my_env

Run the following command to activate your virtual environment try:

$ source ./my_env/Scripts/activate

or for iOS or Linux terminals

$ source ./my_env/bin/activate

In the terminal, round brackets now encapsulate the venv you set up:

(my_env)

** It is possible to follow the file path to the /activate file command within the virtualenv folder. This *venv*, creates an isolated environment for python packages (including Django) and it is located inside your main DjangoGuide folder, where the django_site folder will soon be located.**

$ pip list

pip list will show what requirements are installed into the venv library just created in the project level DjangoGuide folder. To deactive the venv

(my_env)

$ deactivate

With your environment **activated**, run the following command in your terminal to install Django (nested in your newly created environment) with pip:

$ pip install "Django==3.2.2"

pip is a recursive acronym 'pip install python' Now enter in

$ pip list

again to reveal Django, asgiref, pip, pytz, setuptools, and sqlparse modules have been added to the virtual environment.

To start a new project with Django

$ django-admin startproject django_site

Change directory into main django_site.

$ cd django_site/

List files in that directory

$ ls

Notice Django made two directories with that name. Let's now add a single app that will appear in this same directory, along with the inner django_site folder.

$ django-admin startapp mynewapp

*Not yet, but make sure additional apps don't have duplicate names*

$ django-admin startapp AddAnotherApp

**Hint: The mynewapp *folder/directory* does not contain a settings.py file. settings.py is found in the inner "app - level" django_site folder with wsgi.py and asgi.py.

runserver

In the same outer django_site directory, with the manage.py, django_site, and mynewapp folders, enter in the terminal

$ python manage.py runserver

This automatically starts a development server in your local browser at:

127.0.0.1:8000/

If you see an animated green rocketship you have success! Now type ctrl+c to break out of the development server

A class Field creates a table in the model. A table that contains the preset values for multiple fields.
Add the following to models.py in the mynewapp folder: 


from django.db import models
from django.utils import timezone
from django.urls import reverse
from django.contrib.auth.models import User


class PublishedManager(models.Manager):
    def get_queryset(self):
        return super().get_queryset().filter(status='published')


class Post(models.Model):
    STATUS_CHOICES = (
        ('draft', 'Draft'),
        ('published', 'Published'),
    )
    title = models.CharField(max_length=250)
    slug = models.SlugField(max_length=250,
                            unique_for_date='publish')
    author = models.ForeignKey(User,
                              on_delete=models.CASCADE,
                              related_name='blog_posts')
    body = models.TextField()
    publish = models.DateTimeField(default=timezone.now)
    created = models.DateTimeField(auto_now_add=True)
    updated = models.DateTimeField(auto_now=True)
    status = models.CharField(max_length=10,
                              choices=STATUS_CHOICES,
                              default='draft')

    objects = models.Manager() # The default manager.
    published = PublishedManager() # Our custom manager.

    class Meta:
        ordering = ('-publish',)

    def __str__(self):
        return self.title

    def get_absolute_url(self):
        return reverse('blog:post_detail',
                       args=[self.publish.year,
                             self.publish.month,
                             self.publish.day, self.slug])

To get models to appear on the Admin page locate the admin.py in the mynewapp folder. 
from django.contrib import admin
from .models import Post
@admin.register(Post)
class PostAdmin(admin.ModelAdmin):
    list_display = ('title', 'slug', 'author', 'publish', 'status')
    list_filter = ('status', 'created', 'publish', 'author')
    search_fields = ('title', 'body')
    prepopulated_fields = {'slug': ('title',)}
    raw_id_fields = ('author',)
    date_hierarchy = 'publish'
    ordering = ('status', 'publish')
